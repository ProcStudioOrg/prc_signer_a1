# Impeccable Design Guide

## Design Context

### Users
Advogados e equipe jurídica que **já usam certificados digitais A1** mas têm
dificuldade com ferramentas online. O assinador funciona como uma **single/landing
page de marketing de utilidade**: uma página central, pública, que entrega valor
real (assinar documentos com certificado A1) e ao mesmo tempo apresenta o
ProcStudio para quem ainda não conhece o sistema.

Contexto de uso: provavelmente desktop no escritório, mas **precisa ser responsivo
para celular**. O job-to-be-done é assinar um PDF com o certificado A1 de forma
simples, sem fricção técnica.

Momento-chave: **depois de assinar**, mostrar um CTA convidando a conhecer o
sistema ProcStudio ("Conheça o sistema ProcStudio e deixe suas configurações
salvas").

### Brand Personality
**Confiável e profissional.** Sóbrio, seguro, institucional — transmite
credibilidade jurídica. Três palavras: **confiável, profissional, simples.**
A interface deve reduzir a ansiedade de quem não é técnico, deixando claro cada
passo. Nada de jargão ou ruído visual.

### Aesthetic Direction
- **Sistema de design ProcStudio** (já presente em `src/app.css`):
  - Navy: `--navy #0f1a3e`, `--navy-dark #0a0e2a`
  - Azul de marca: `--blue #0088FF`, hover `#006dd9`, light `#e8f4ff`, border `#b3d9ff`
  - Escala de cinzas: `--gray-50 #f7f9fc` … `--gray-700 #3d4f63`
  - Fonte: system stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto`)
- Layout em **card central** com borda-topo azul, sombra suave, cantos arredondados.
- **Light + dark mode** (respeitar `prefers-color-scheme`). Os tokens devem
  funcionar nos dois temas — definir variáveis para superfícies, texto e bordas
  que invertem no dark.
- Favicon oficial: `/Users/brpl/code/ProcStudio/ProcStudio-Docker/customer-frontend/public/favicon.ico`.
  Logos disponíveis em `ProcStudio-Docker/frontend/src/assets/` e o símbolo já em uso no repo.
- Anti-referência: nada que pareça "ferramenta técnica crua" ou genérica de TI.
  Deve parecer um produto jurídico polido e digno de confiança.

### Design Principles
1. **Confiança acima de tudo.** Cada elemento deve reforçar segurança e
   credibilidade — é assinatura de documento jurídico. Estados claros de
   sucesso/erro, sem ambiguidade.
2. **Simplicidade para não-técnicos.** Fluxo óbvio, um passo de cada vez,
   linguagem clara em português. Reduzir decisões e jargão.
3. **Mobile-first responsivo.** Funciona bem no celular e no desktop; o card
   central reflui sem quebrar.
4. **Light + dark mode via tokens.** Cores por variável CSS semântica, nunca
   hardcoded, para os dois temas conviverem.
5. **Converter visitantes.** A página é vitrine do ProcStudio: CTA pós-assinatura
   conduzindo ao sistema completo, sem atrapalhar a tarefa principal.

### Accessibility
- **Meta: WCAG AA** para contraste de cor (texto e componentes) nos dois temas.
- **Sem requisito de navegação por teclado** — decisão explícita do produto: é um
  sistema simples e não faz sentido investir nisso aqui.
- Respeitar `prefers-reduced-motion`. Labels associados a todos os inputs do
  formulário (já presente).
